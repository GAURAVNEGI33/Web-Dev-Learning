# Git & GitHub Learning

In this module, I learned the basics of Git and GitHub. I understood how Git helps track changes in a project, while GitHub allows me to store my code online, collaborate with others, and showcase my work.

---

# What I Learned

## What is Git?

Git is a version control system that keeps track of changes made to a project. It allows me to save different versions of my code and makes it easy to manage updates.

---

## What is GitHub?

GitHub is a platform where Git repositories are hosted online. It allows me to back up my projects, share them with others, and maintain a portfolio of my work.

---

## Creating a Repository

I learned how to:

- Create a new repository.
- Add a README file.
- Organize project files.
- Push local projects to GitHub.

---

## Basic Git Workflow

The basic workflow I practiced is:

```bash
git init
git status
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin <repository-url>
git push -u origin main
```

I also learned that after the first push, updating the repository usually requires:

```bash
git add .
git commit -m "Describe the changes"
git push
```

---

## Useful Git Commands

Some commands I practiced include:

- `git init`
- `git status`
- `git add`
- `git commit`
- `git push`
- `git pull`
- `git clone`
- `git log`

## README Files

I learned that every repository should include a **README.md** file because it explains:

- What the project is about.
- What was learned.
- How to use the project.
- Any important information for others visiting the repository.

---

## GitHub as a Portfolio

I understood that GitHub is not only used for storing code but also for showcasing projects and tracking my learning journey.

Keeping repositories organized with meaningful commit messages makes my profile look more professional.
