# HTML Learning – Building My Resume

In this session, I learned how to structure a webpage using **Semantic HTML** by building my own resume. Instead of relying only on `<div>` elements, I used meaningful HTML tags to create a clean and well-organized layout.

This helped me understand how HTML provides structure before adding any CSS styling.

---

# What I Learned

## Semantic HTML

I learned that semantic elements describe the purpose of different sections of a webpage. They make the code easier to read, improve accessibility, and help search engines understand the content better.

---

## Semantic Elements I Used

### `<header>`
Used for the top section of my resume, including my name and introduction.

### `<main>`
Wrapped the main content of the resume.

### `<section>`
Separated different parts of the resume such as Education, Skills, Projects, and Experience.

### `<article>`
Used for individual content blocks where appropriate.

### `<aside>`
Created a sidebar containing personal information, contact details, skills, and languages.

### `<footer>`
Added the footer section for additional information.

---

# Resume Structure

While building the resume, I divided the page into meaningful sections instead of placing everything inside `<div>` elements.

The structure looked something like this:

```html
<header></header>

<main>
    <aside></aside>

    <section>
        <article></article>
    </section>

    <section>
        <article></article>
    </section>
</main>

<footer></footer>
```

# Key Takeaways
